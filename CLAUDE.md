## プロジェクト概要

- **tezukai**: 収入と各種控除から税金を計算する個人向けの税金シミュレータ（公開 Web アプリ）。
- **目的**: 年収と控除を入力すると、所得税・住民税・社会保険料を計算し、**年収 → 手取り**まで可視化する。あわせてふるさと納税の控除上限額や住宅ローン控除などの試算もできる。
- **非目標**: あくまで**概算ツール**。確定申告書類の作成／税理士的な個別助言ではない（免責表示を置く）。**個人情報は入力・保存しない**（全てブラウザ内計算、サーバー送信なし）。
- **公開前提**: 個人情報を扱わない静的サイトとして公開する。デプロイ先は GitHub Pages を想定。

## 対象者（モード切替）

- **フェーズ1（優先）**: 給与所得者（会社員・パート）。給与所得控除を使う。
- **フェーズ2**: 個人事業主（事業所得）。モード切替で対応。
- 計算ロジックは**最初から両モード拡張前提で分離設計**する。

## 計算スコープ（段階的に実装）

| 区分 | 内容 | 優先 |
|---|---|---|
| コア | 給与所得控除 → 所得控除（基礎・社保・配偶者・扶養 等）→ 課税所得 → 所得税 → 住民税（所得割＋均等割）→ 社会保険料 → **手取り** | ★最優先 |
| 拡張 | ふるさと納税の控除上限額シミュレーション | ★高 |
| 拡張 | 住宅ローン控除 | 中 |
| 拡張 | 医療費控除・iDeCo（小規模企業共済等掛金）・生命保険料控除 等 | 中 |

## 必須プロセス（最重要）

1. **TDD（テスト駆動）**: 計算ロジックは Vitest で Red→Green→Refactor。税額は**公式資料に基づくケース**でテストを書いてから実装する。純粋関数のドメイン層を中心に検証する。
2. **税率・控除額は年度別データとして分離**: 定数（税率テーブル・各控除額・上限）は `data/taxTables/<年度>.ts` のように**ロジックから分離**する。税制は毎年変わるため、数値だけ差し替え・検証できる構造にする。数値を入れる/変える際は**出典（国税庁・総務省等）をコメントで残す**。正確性が命。
3. **個人情報・シークレットを絶対にコミットしない**。本アプリは原則どちらも持たないが、`.env*` は `.gitignore` 済み。

## コマンド

```bash
npm install        # 依存インストール
npm run dev        # ローカル開発サーバー（Vite）
npm run test       # Vitest（計算ロジックのテスト、1回実行）
npm run test:watch # Vitest（watch モード）
npm run typecheck  # 型チェックのみ（tsc -b --noEmit）
npm run build      # 本番ビルド（tsc -b → vite build → dist/）
npm run preview    # 本番ビルドのローカルプレビュー
```

公開: `main` への push で GitHub Actions（`.github/workflows/deploy.yml`）が
ビルド→テスト→GitHub Pages へ自動デプロイする。本番ビルドのベースパスは
`vite.config.ts` で `/tezukai/`（リポジトリ名）に設定。

## アーキテクチャ早見表（予定）

- `core/` … UI 非依存の**計算ドメイン層**（純粋関数）。所得区分・各控除・所得税/住民税/社会保険料・手取り計算、ふるさと納税上限、住宅ローン控除など。Vitest でテストする中心。
- `data/taxTables/` … 年度別の税率・控除の定数テーブル（出典コメント付き）。
- `app/` … React の UI（入力フォーム・結果表示）。`core/` を呼ぶだけにし、計算は UI に書かない。

## 設計上の約束

- **計算は `core/` の純粋関数**に集約し、UI（`app/`）から完全分離する。
- UI はすべて**日本語**。
- Secrets / バックエンド無しで動く（完全クライアントサイド）。
- 概算であることと免責を UI に明示する。
